# Reports

This folder collects past branching policy documents and advisory outputs produced by `branching-strategy-guardian`.

Each time the Angel produces a branching policy document for a team, the output may be archived here as `YYYY-MM-DD-{repo-or-team-slug}-branching-policy.md`.

## Report format

Each report should include:
- The team context (size, product type, release cadence)
- The model recommendation and rationale
- The migration plan (if applicable)
- The feature flag verdict
- Any open questions routed to `github-repo-health-guardian` or `devops-guardian`
